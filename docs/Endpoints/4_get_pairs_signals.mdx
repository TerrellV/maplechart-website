# GET • pairs/signals

Get latest recorded signals for set of pairs.

## Path Parameters

No path parameters accepted

## Query Parameters

| Parameter  | Required | Example                   | Description                                                                                                                                             |
| :--------- | :------: | :------------------------ | :------------------------------------------------------------------------------------------------------------------------------------------------------ |
| pairs      |    NO    | btc-usd, eth-btc, eth-usd | List of _**comma-separated pairs**_ to fetch latest signals for. The `pairs` param cannot be specified with `quote` param. You must use one or neither. |
| quote      |    NO    | 2018-01-01                | Quote currency to use for selecting trading pairs. The `quote` param cannot be specified with `pairs` param. You must use one or neither.               |
| next_token |    NO    | 234asdf5!23==             | Used for paginating through results. See pagination guide for more info                                                                                 |

## Response

The `/pairs/signals` endpoint will return a json document with a data property.
The data property will be an array of objects. Each object contains the most recently recorded signals for a trading pair.

### Pagination

Because the amount of data available from this endpoint is large, it is common for responses to be paginated.
See the [pagination guide](../pagination) to learn how to handle paginated results.
